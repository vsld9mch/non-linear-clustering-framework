##  Project Overview
This project is a comprehensive benchmarking and research framework that explores the structural and geometric limitations of traditional unsupervised clustering algorithms ($K\text{-Means}$ and Agglomerative Hierarchical Clustering). 

To overcome the inherent geometric constraints of traditional methods on non-convex shapes, the framework introduces a **hybrid Machine Learning pipeline**. By feeding spatial coordinates combined with unsupervised cluster assignments into a multi-layer Artificial Neural Network ($ANN$), the deep learning model successfully learns complex non-linear decision boundaries, achieving near 100% error correction.

---

##  Framework Architecture & Workflow

### 1. Synthetic Data Generation & Spatial Geometry
The pipeline evaluates algorithms across three distinct synthetic datasets generated via `scikit-learn` to isolate specific geometric properties:
* **Spherical Clusters:** Linearly separable, convex spatial distributions (`make_blobs`).
* **Non-Linear Moons:** Intertwined, non-convex structures (`make_moons`).
* **Concentric Circles:** Nested, non-linear ring geometries (`make_circles`).

All generated data points are structured into `Pandas` DataFrames and backed up as local `.csv` files to mimic standard database pipeline extraction.

### 2. Unsupervised Clustering Benchmarking
* **K-Means Clustering:** Implemented using `k-means++` initialization to benchmark centroid-based partitioning.
* **Hierarchical Clustering:** Implemented via `AgglomerativeClustering` using the `ward` linkage method and Euclidean distance metrics.
* **Linkage Stress-Testing:** A comparative loop evaluates how different mathematical linkage criteria (`ward`, `complete`, `average`) alter the silhouette coefficient across varying geometries.
* **The Silhouette Trap:** The framework quantitatively proves why the **Silhouette Score** can be highly misleading, as it mathematically favors spherical/convex clusters even when the visual partition on non-linear shapes is completely incorrect.

### 3. Deep Learning Error Correction (Hybrid ANN)
To resolve the structural failures of unsupervised methods on the non-linear `Moons` dataset:
* **Feature Engineering:** Features are engineered by horizontally stacking the original 2D coordinates with the "flawed" cluster labels generated by $K\text{-Means}$.
* **Data Pipelines:** Data is split into training/testing sets ($80/20$) and scaled via `StandardScaler` to optimize gradient descent performance.
* **ANN Architecture:** A multi-layer `Sequential` model built in `TensorFlow/Keras` featuring:
  * An input layer matched to the hybrid feature space.
  * Hidden layers with **ReLU** non-linear activation functions to map complex boundaries.
  * A **Dropout** layer ($0.1$) to strictly enforce regularization and prevent overfitting.
  * A final single-unit layer with a **Sigmoid** activation function for definitive binary classification.

---

##  Core Conclusions & Findings
1. **Convex Domination:** Traditional algorithms ($K\text{-Means}$ and Hierarchical with Ward linkage) excel perfectly at detecting spherical, convex clusters.
2. **Geometric Blindness:** Both unsupervised methods completely fail when mapping complex, intertwined non-linear structures (`Moons` and `Circles`), cutting the shapes linearly rather than following their natural curves.
3. **The Hybrid Solution:** By utilizing Deep Learning non-linear mappings, the $ANN$ uses the baseline clustering insights as an anchor and corrects the partition errors, elevating test prediction accuracy to **~100%**.
